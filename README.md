# PPPI

**Тема:** Командна розробка в Git  
**Мета роботи:** Отримати навички спільної роботи над проєктом за допомогою Git. Навчитися створювати гілки, працювати з пул-запитами та вирішувати конфлікти при злитті.

## Короткий опис проєкту
Цей репозиторій створено в рамках лабораторної роботи для відпрацювання командної розробки з використанням Git та GitHub. Team Lead(Бурковська Аделіна) створює початкову структуру класів та оголошує методи, інші учасники реалізують свої частини в окремих гілках.

## Інструкції для роботи в команді
1. **Team Lead(Бурковська Аделіна)**
   - Створює репозиторій на GitHub.
   - Додає `README.md` та `.gitignore`.
   - Додає інших студентів як collaborators (Settings → Manage access).

2. **Кожен учасник**
   - Створює нову гілку для роботи: `feature-<прізвище>` (наприклад `feature-ivanov`).
   - Виконує роботу локально, робить `git add` та `git commit`.
   - Пушить гілку на GitHub: `git push -u origin feature-<прізвище>`.
   - Створює Pull Request у main з описом змін.

3. **Перегляд і злиття**
   - Team Lead залишає коментарі, після затвердження зливає у `main`.

4. **Робота з конфліктами**
   - При виникненні конфлікту — вирішуємо вручну, робимо commit і завершуємо merge.
